# push-reactive —— 推送式反应数据流框架（同步级联 + 节点观察）

把一张 `tok` 依赖图当作**反应式电子表格**：设一个源格，沿图**同步级联**重算所有下游派生格，
派生格的值一落定，挂在格上的**节点钩子 `exec`** 即时反应。**无队列、无线程**——求值与传播全在
设值线程内同步完成。本模板与 [workflow-graph](../workflow-graph/)（拉取式）成**对照对**：
同一张图，两种执行模型。

## 拉取式 vs 推送式（同图，反执行）

| | 拉取式（workflow-graph） | 推送式（本模板） |
|---|---|---|
| 触发 | 源 set → dep 路由**入队** | 源 set → 沿图**同步级联** |
| 计算时机 | worker 经 `back` 拉取（`exec`）后才跑 | 级联中即时 `combine` |
| 并发 | 多线程 + 环形缓冲队列 | 单线程同步 |
| 求值/传播 | 解耦（快慢级分离） | 耦合（即时） |
| 节点钩子 | `exec`（拉取消费） | `exec`（推送反应） |
| 变更检测 | **危害**（同值丢帧，需 `pulse` 逃生） | **特性**（同值截断 = 反应式去抖/记忆化） |

## 职责划分（dep 管路由，node 管函数）

与 workflow 同一哲学，同步形态：

- **node（点）= 函数实现**：单输入派生格的**公式**写在该格的 `combine` 体里
  （`this->input` 即上游值，`return` 新值）——格即函数。多输入扇入格的公式天然落在 `follow`。
- **dep（边）= 关系路由**：单输入边的 `follow` 只把上游值**前推**给下游格
  （`t->set(s->get())`），触发下游 `combine` 重算——边只连线、不算公式。
- **exec（节点反应）= 副作用/观察点**：格值落定后于**锁外**唤起，做 `combine` 不能做的
  副作用（产出、统计、日志、外部推送）。`combine` 须纯（锁内只算值），副作用归 `exec`；
  本模板通过 `form N, (0:@), &N_n, observe` 绑定。

## 运行

```sh
./compiler/build/scc templates/push-reactive/reactive.sc
```

输出：先打印整张反应图的烘焙度量表，再以 8 个读数同步级联，最后打印各观察格的确定性统计。

## 内置反应图（线性级联 + 三处观察）

```
sensor ─► norm ─► level ─► warn
```

- `sensor` 纯源（`depth=0`）：`set` 即原值。
- `norm = in/10`、`level = min(in, 5)`（饱和 clamp）、`warn = in >= 4 ? 1 : 0`——公式各写在格的 `combine`。
- `norm/level/warn` 各挂一个 `exec` 观察钩子，累计「该格**实际变更**」的统计。

## 反应式记忆化（变更检测在推送场景是特性）

`set` 仅在「新值≠原值」时落值并传播。推送反应式里这正是**辉光抑制/去抖**：上游变了但某格
重算后值不变（如 `level` 饱和到 5），级联就在该格**自然截断**，下游不做冗余重算、`exec`
不被无谓唤起。驱动 `sensor` 读数 `10,20,…,80`，输出确定性地体现级联随饱和**逐级衰减**：

| 格 | 公式输出序列 | 实际变更次数 | 观察累计 |
|---|---|---|---|
| `norm`  | 1,2,3,4,5,6,7,8 | **8**（每次都变） | 36 |
| `level` | 1,2,3,4,5,5,5,5 | **5**（饱和后截断） | 15 |
| `warn`  | 0,0,0,1,1,1,1,1 | **1**（仅跨阈一次） | 1 |

这与拉取式流水线相反：那里「每帧皆事件、相同值也不可丢」，同值抑制会丢帧，需调用点侧
`t->pulse(v, tag)` 强制刷新。**同一机制，两种场景两种语义。**

## 框架自省（编译期烘焙，运行时 O(1) 查表）

与 workflow 共享同一套烘焙度量（`depth/critical/slack/batch/reach/checkpoint`），由编译器在
编译期把每个格的结构度量算成常量烘焙进二进制，运行时零图遍历。推送场景尤其可用 `reach()`
（变更波及面）预判一次源 set 的最大重算规模、用 `checkpoint()` 设记忆化缓存边界。

## 扩展

详见 `reactive.sc` 文末「扩展指南」，要点：

1. **加一单输入派生格**：`tok N`（公式写 `combine`）+ 一条**单行前推** `dep…map`；要观察则
  加 `var N_n: cnode` + `cn_init` + `form N, (0:@), &N_n, observe`。
2. **扇入格**（公式依赖多输入）：公式天然落在 `follow`（需读全部输入），格本身无 `combine` 体。
3. **「每事件皆须传播」的逃生门**：推送图里纯事件格可在 `combine` 内 `return tok_modified()` 强制传播；
  若是调用点侧要「同值也不丢」，请用 `t->pulse(v, tag)`。这通常意味着你要的是拉取/迭代语义。
4. **接 MT**：`exec` 在锁外唤起、天然 MT 安全；id 以 `/` 开头即启 seqlock 值同步。

见 `builtins/tok/`（tok 内核）与 `syntax.md` §21（tok 体系）。
