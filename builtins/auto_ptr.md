# 自动指针 `T@`：双向引用图与释放点验证机制

> sc 语言最精华、也最难的部分。本文是**定稿规格 + 机制流程 + 全场景示例**，
> 既面向用户讲清怎么用，也作为编译器实现的对照基准（避免后续遗忘）。

---

## 0. 设计目标与边界

**目标**：在**不引入** Rust 所有权/生命周期标注、**不引入** GC 的前提下，让 sc 能：

1. 自动释放孤立的堆对象（ARC 标准行为）；
2. 在**释放点**检测出 use-after-free（悬挂）与「释放时引用关系不完备」（出边未清理），并定位到源码行。

核心承诺是 op.sc「引用与释放机制」原则 2 的精确化：

> **确保根节点在释放点满足「完全孤立」状态；保证正确且错误可追踪，而非完全替代手工内存管理。**

**明确不做**（这些是设计边界，不是 bug，写进文档当承诺）：

- **不递归连带释放**：堆对象归零自动 free 自身，但**不**像 Rust 那样按所属权级联释放子节点。这里**不区分所属权**，子节点各自通过同一套 ARC 计数独立释放。
- **不做反向失效**：目标死时不会自动把指向它的指针置 nil（那需要 referrer 侵入链，即被否掉的复杂结构）；目标提前死 → 在释放点**报错**，而非静默修复。
- **不保证无泄漏**：运行时别名造成的环引用，须用户手工断边；忘记释放照漏（同 C）。
- **转裸即退出**：`T@` 一旦显式转裸 `T&`，立即退回纯 C 语义，转裸之后不可检测。

## 1. 核心模型：指针是双向边

一次指针赋值 `A.p = &B` 物理上是一条从 A 出发的边，但它在生命周期上**同时约束两端**：

- **B 不能比这条边先死** —— 否则 `A.p` 悬挂。（约束作用在 B 上）
- **A 不能丢下这条边就死** —— 否则 B 泄露。（约束作用在 A 上）

传统方案只记单向，所以只能保证一头：

| 方案 | 记账方向 | 能做 | 不能做 |
|---|---|---|---|
| C++ `shared_ptr` | 只记入边（B 被几人指） | 自动释放 B | 抓不到 A 持有边的悬挂 |
| Rust | 借用实体化、编译期验证两头 | 全保证 | 概念重、数据结构转不了 C |
| **sc `T@`** | **一条边两端各记一次** | 自动释放 + 双向悬挂/泄露检测 | 不级联、不反向失效 |

所以 sc 对一条边在**两个端点各记一次账**：

| 计数 | 含义 | 由谁维护 | 谁违反它 |
|---|---|---|---|
| `in`（入边数） | 多少胖指针指向本对象 | 胖指针的 `tar` | `in>0` 时释放 → **悬挂** |
| `out`（出边数） | 本对象持有多少胖指针 | 胖指针的 `own` | `out>0` 时释放 → **泄露/未清理** |

**释放不变式**：

```
对象可释放  ⟺  in == 0  且  out == 0  ⟺  完全孤立节点
```

`in` 与 `out` **必须分开两个计数**（不可合并为单一 ref）：
- 释放点要区分「还有人指着我（悬挂）」与「我还指着别人（未清理）」两种不同错误；
- 只有独立的「`in→0`」事件，才能触发对**堆内部节点**的检查（见 §5 round-4 洞）。

## 2. 类型系统：`T&` 裸指针 vs `T@` 胖指针

两条**完全独立**的路径，胖指针 opt-in，谁用谁付内存/时间：

```sc
var raw: T&        # 裸指针，8 字节，纯 C 语义，不追踪（现状不变）
var ptr: T@        # 胖指针，24 字节，参与引用图与释放点验证
```

转换规则：

- **裸 → 胖：禁止隐式**。裸指针没有 `tar/own` 信息，无法安全升级为胖指针。
- **胖 → 裸：显式允许** `r: & = ptr`（取 `ptr.p`）。转裸后**不再维护**目标引用计数，
  用于兼容 C ABI；用户自行确保该裸指针不被长期占有。**这是 opt-out 的唯一出口。**
- 胖指针首成员 `p` 即裸指针，`&fat == &fat.p`，地址值一致，可直接强转（值一致性）。

## 3. 内存布局

### 3.1 对象头 `sc_ref`（位于对象实体之前）

```c
/* 标准构建 */ typedef struct { int32_t in, out; } sc_ref;
/* 追踪构建 */ typedef struct { int32_t in, out; const char* site; } sc_ref;  /* site=分配点 file:line */
```

### 3.2 堆对象（`T()` 分配）

```
[ sc_ref 头 | pad 对齐到 alignof(T) | ===== T 实体 ===== ]
                                      ↑ T() 返回这里（用户拿到的指针）

hdr(p) = (sc_ref*)((char*)p - HDR)，  HDR = 对齐(sizeof(sc_ref), alignof(T))
```

### 3.3 胖指针

```c
typedef struct { T* p; int32_t* tar; int32_t* own; } T_fat;
```

- `p`   —— 目标对象地址（首成员，可强转裸指针）。
- `tar` —— 目标对象 `sc_ref.in` 的地址（维护入边）。
- `own` —— 持有本指针的对象 `sc_ref.out` 的地址（维护出边），用哨兵值标识特殊归属：

| `own` 值 | 含义 |
|---|---|
| `&owner.out` | 普通：本指针属某对象，出边记在该对象 out 上 |
| `SC_OWN_ROOT = (int32_t*)-1` | 栈/全局根指针，域退出自动拆 |
| `SC_OWN_RAW  = (int32_t*)-2` | 经裸 base 访问/显式转裸，不追踪 |
| `NULL` | 未初始化，首次访问按 §4.3 规则初始化 |

> `tar == NULL` 表示目标无头（指向 C 内存/栈未注入）：不计入边、不检测该目标悬挂（边界）。

## 4. 引用图维护规则

### 4.1 赋值（绑定）与清除（解绑）

```sc
owner.p = ⌖B      # 绑定：先拆旧边(旧目标.in--, owner.out--)，再绑新边(B.in++, owner.out++)
owner.p = nil     # 解绑：B.in--，owner.out--
```

> ⚠ **重新赋值必先拆旧边**（最易漏的洞）：`p = q` 时 p 已指旧目标，必须先让
> `旧目标.in--`、`owner.out--`，再绑新目标，否则旧目标 `in` 永降不下来 → 释放点假阳性。

解绑后若某对象 `in` 触 0 → 触发它的释放点（§5）。

### 4.2 `&` 取址：传染性 + tar/own 归属

- **传染性**：对胖指针目标的（递归）子成员取址 → 结果**也是**胖指针。
- **tar 来自访问路径上最后一个胖指针 hop**：
  - `&base->a.b.c`（base 胖）→ 结果 `tar = base.tar`（a.b.c 住在 base 的目标里，共享其 in）。
  - `&base->a.b.ptr->c.d`（ptr 是胖子成员）→ 结果 `tar = ptr.tar`（c.d 住在 ptr 的目标里）。
- **own 由接收者（赋值方）按 §4.3 规则定**。
- 绑定时**双方各 +1**（目标.in++、接收者所属对象.out++），构成双向连接。
- **& 目标是普通栈/全局域变量**（不在任何胖目标内）→ 在其所在域起始**自动注入 `sc_ref` 头**，
  结果 tar 指向它。这样「借出栈变量地址」也纳入 in 计数，从而能在该栈变量退域时检测外部悬挂。

### 4.3 own 身份的确定（首次访问 `own==NULL` 时初始化）

| 胖指针位置 | own |
|---|---|
| 栈/全局域变量（根指针） | `SC_OWN_ROOT(-1)`，域退出自动拆 |
| 子成员，经**胖** base 访问 `base->...->p` | base 最后一个胖 hop 所指对象的 `out` 地址 |
| 子成员，经**裸** base 访问 | `SC_OWN_RAW(-2)`，退化不追踪（见 §7.4，实为禁止写） |
| `&` 取址结果赋给的接收者 | 按接收者自身位置递归套用本表 |

## 5. 释放点与验证（核心）

**释放点**在以下时刻产生，每个释放点都执行同一套验证：

```c
/* 标准断言（追踪构建附带 site 源码定位） */
static void sc_release_check(sc_ref* r, const char* who){
    if (r->in  != 0) ERROR("悬挂：%s 释放时仍被 %d 个引用指向", who, r->in);
    if (r->out != 0) ERROR("未清理：%s 释放时仍持有 %d 个出向引用", who, r->out);
}
```

释放点的三种触发：

- **(a) 域退出**：自动拆域内每个**栈根指针**的边（目标.in--），随后对域内每个栈对象
  `sc_release_check`。
  **两阶段固化：先全部拆边，再全部断言**（否则同域互指会互相顶住而误报）。前提目标一致。

- **(b) 任意堆对象 `in→0`**（ARC 触发点）：
  - `in==0 && out==0` → **自动 `free`**（ARC 标准行为；**不**递归释放子节点）。
  - `in==0 && out>0`  → **报错**：「释放 X 时仍持有未清理的出向引用」。

- **(c) 用户显式 `free` 堆对象**：free 前 `sc_release_check`。

> **round-4 洞（已修复）**：`root@→A@`，`A.p→B`，用户 `root=nil` 使 `A.in→0`。
> 若只在域根退出检查，堆内部节点 A 无人查 → A 及其出边 B 静默泄露。
> 靠「`in→0` 即释放点」堵死：`A.in→0` 时 `A.out=1` → 报「释放 A 时仍持有 `A.p→B`」。
> 用户须先 `A.p=nil` 再断 `root`，此时 `A.in→0 && A.out==0` → 自动 free A，B 同理归零自动释放。
> **不级联** = A 归零不会去主动 free B；是 A.p 解绑使 B.in-- 让 B **自己**归零才 free。

## 6. 堆分配 `T()`

- 仅当**赋值目标是 `T@`** 时，`T()` 才分配 `sc_ref` 头（`in=0, out=0`，追踪构建记 site）；
  否则退化为普通分配（无头，纯 C 指针）。
- `T@` 目标必须**地址稳定**（带头堆对象 / 栈 / 全局）；按值塞进会 realloc 的容器槽的对象
  不能当胖目标（搬家后 tar 成野指针）。

## 7. 全场景示例

### 7.1 栈根指针 → 堆对象（最简，正确）
```sc
fnc f::
    var p: T@ = T()      # p.own=ROOT；新堆对象 obj.in=1
    # ... 用 p ...
    return               # 域退出：拆 p 边 → obj.in→0 && out==0 → 自动 free ✓
```

### 7.2 堆对象持有出边 `A.p→B`（须手工拆，否则报错）
```sc
var a: T@ = T()
var b: T@ = T()
a->child = b             # B.in++（=2：b根 + a->child），A.out++（=1）

# 错误：直接结束 a
# a = nil                # A.in→0 但 A.out=1 → 报"释放 A 时仍持有 a->child→B"

# 正确：自底向上拆
a->child = nil           # B.in--，A.out--（A.out→0）
a = nil                  # A.in→0 且 A.out==0 → 自动 free A ✓
b = nil                  # B.in→0 → 自动 free B ✓
```

### 7.3 借用比目标活得久（悬挂被捕获）
```sc
var x: T@
do
    var local: T@ = T()
    x = &local->field    # x.tar = local目标.in；local目标.in++
while false              # 内层域退出：拆 local 根边，但 x 仍持入边 → in>0 → 报悬挂
```

### 7.4 经裸 base 访问胖成员（**禁止写**）
```sc
def Node: { child: T@ }
var raw: Node& = ...     # 裸 base
# raw->child = b         # 编译报错：禁止经裸指针写胖成员
```
原因：无 owner 的 out 可挂。bump `B.in` 则永不对称 dec → 假阳性；不 bump 则漏报真悬挂。
**规则**：含 `T@` 成员的结构体，访问其胖成员必须经 `T@` base（或该成员声明为裸 `T&`）。

### 7.5 环 / 自引用（释放点报错，须手工断边）
```sc
var a: T@ = T()
a->next = a              # 自引用：A.in++（=2），A.out++（=1）
# a = nil                # A.in=1（a->next 仍在） → 报错
a->next = nil            # 先断环：A.in--，A.out--
a = nil                  # A.in→0 && out==0 → 自动 free ✓
```
静态字面自引用（在 a 自己方法内 `&a`）可识别并特殊处理；**运行时别名造成的环**静态测不出，
须用户手工断边 —— 这是 §0 承诺边界。

### 7.6 数组
```sc
var arr[10]: T@          # 退域遍历 10 个 dec；sizeof(arr)=10*24，与 C 指针数组不同
```

### 7.7 函数传参 / 返回

跨函数边界是 own/tar 记账的关键场景，分**入参（借用语义）**与**返回（移动语义）**两类，规则不同。

#### (1) 入参：按值传 `T@` = 借用（一对 inc/dec）

形参是被调用方栈帧里的一个**新胖指针根**（`own = SC_OWN_ROOT`），它与实参在调用期间**同时**指向同一目标：

```sc
fnc use: x: T@           # 形参 x 是 callee 帧的根指针
    # 调用期间 目标.in 比平时多 1（caller 实参 + callee 形参）
    return               # callee 退帧：拆 x 边 → 目标.in--

fnc caller::
    var a: T@ = T()      # 目标.in = 1
    use(a)               # 进入: 目标.in++ (=2)；退出: 目标.in-- (=1)。net 一对 inc/dec
    return
```

- **绑定时机**：进入 callee 时把形参当根指针绑定（`目标.in++`）；callee 作用域退出（`return`/
  出口）时解绑（`目标.in--`）。净效果是**一对 inc/dec**，计数走 `operand` 原子指令则原子、跨线程安全。
- **为什么要计数**：① 调用期间顶住目标，防止 callee 内部把它释放导致 caller 实参悬挂；
  ② 若 callee 把形参**存进更长命的出边**（如 `g->slot = x`），那是一次独立的 bind
  （`目标.in++` 记在 `g` 的 out 上），与形参借用的那对 inc/dec 正交，目标因此被正确续命。
- **纯借用优化（未来）**：若静态可证形参在 callee 内**不逃逸**（不被存入任何出边、不被 `&` 再借出），
  这对 inc/dec 可安全省略（目标在同步调用期间本就不会死）。首版可先全计数，后续按逃逸分析优化。
- **传给裸 `T&` 形参**：实参 `T@`、形参 `T&` 时，按调用点**隐式转裸借用**处理——传 `x.p`、不计数，
  callee 拿到纯裸指针（等价 §7.8 的 `: &`）。此后 callee 侧不可追踪（边界）。

#### (2) 返回：`return p` = 移动（move-out，不在 callee 拆边）

返回一个 `T@` 是**所有权转移**，不是借用：callee **不**对被返回的局部解绑（不 `目标.in--`），
把这条入边**整体移交**给调用方接收者：

```sc
fnc make: T@             # 返回类型 T@
    var p: T@ = T()      # 目标.in = 1
    return p             # 移动：不拆 p 边（目标.in 保持 1），把边移交出去

fnc caller::
    var x: T@ = make()   # x 接管该边：目标.in 不变(=1)，仅置 x.own 并 x 所属.out++
    return               # 域退出：拆 x 边 → 目标.in→0 → 自动 free ✓
```

- **callee 侧**：被返回的局部走移动路径，**跳过**它在作用域退出时的那次 `目标.in--`
  （否则一进一出归零会把刚要返回的对象误释放）。
- **caller 侧接收 `x = make()`**：入边已存在（移动而来），故**不再 `目标.in++`**；只需按 x 的位置
  确定 `x.own` 并对 x 所属对象 `out++`（§4.3）。即移动只重挂**出边侧**记账，入边计数守恒。
- **返回值被丢弃**（`make()` 单独成句、不接收）：这条移动来的入边没有新主 → 编译器在**丢弃点
  自动解绑**该临时（`目标.in--`），避免目标 in 永不归零的假阳性。等价于"即创建即释放"。
- **链式 `f(make())`**：`make()` 的返回临时作为实参传入，按入参借用规则在 `f` 调用结束后由编译器
  解绑该临时（与丢弃同理），不泄漏。

### 7.8 转裸 opt-out（兼容 C ABI）
```sc
var p: T@ = T()
some_c_api(p: &)         # 显式转裸传 C；此调用不维护计数；用户保证 C 侧不长期占有
```

## 8. C 转译方案（标准 C99，无扩展、无 libuv）

```c
typedef struct { int32_t in, out; } sc_ref;            /* 追踪构建多一个 const char* site */
typedef struct { T* p; int32_t* tar; int32_t* own; } T_fat;

#define SC_OWN_ROOT ((int32_t*)-1)
#define SC_OWN_RAW  ((int32_t*)-2)

static inline void sc_bind(T_fat* f, T* tgt, sc_ref* tr, int32_t* ow){
    f->p = tgt; f->tar = tr ? &tr->in : 0; f->own = ow;
    if (f->tar) (*f->tar)++;
    if (f->own > (int32_t*)0) (*f->own)++;             /* 跳过 ROOT(-1)/RAW(-2) 哨兵 */
}
static inline void sc_unbind(T_fat* f){
    if (f->tar && --(*f->tar) == 0) sc_on_zero_in(f);  /* in→0：自动 free 或报 out>0 */
    if (f->own > (int32_t*)0) (*f->own)--;
    f->p = 0; f->tar = 0;
}
/* 域退出：①先对域内全部根指针 sc_unbind  ②再对域内栈对象 sc_release_check */
```

## 9. 与现有机制的衔接

| 现有机制 | 对接方式 |
|---|---|
| `T()` / `init` / `drop` | `T@` 目标的 `T()` 带头分配；归零自动 free 前若有 `drop` 先调 |
| `operand` 原子指令 | `in/out` 计数增减直接用 `sc_inc`/`sc_get_and_dec`，跨线程即 Arc 语义 |
| slice `_self` 回指 | 与 own 回链同构，可复用「反查本体」代码思路 |
| `chain`/容器 | 维持「不拥有元素」 = 存裸 `T&`（不计 in/out），避免环；对象生命周期由别处 `T@` 槽掌管 |
| 现有静态检查（局部地址逃逸、nil 边界） | 作为编译期第一道防线保留；本机制做运行时释放点兜底 |
| `run`/`async` 跨线程 | 跨线程共享 `T@` 用原子计数即正确 |

## 10. 构建选项（独立开关，不绑定 release/debug）

- 引用追踪/释放点验证是**独立编译选项**（如 `scc --check=ref` / `-DSCC_REF_CHECK`），
  不与 release/debug 绑定。
- **开启**：`sc_ref` 带 `site`，插入 `sc_release_check` 全部断言，报错带源码定位
  （「对象 alloc@a.sc:12 释放时仍被 p（声明于 c.sc:7）引用」）。
- **关闭（默认）**：ARC 计数与自动 free 仍在（功能）；栈释放点断言可编译为 no-op 省开销。
- 极端关闭（纯 C）：若工程不要 ARC，可整套计数编译掉，`T@` 退化为裸指针布局（仅调试期用）。

---

## 11. 设计边界速查（向用户的承诺）

| 能 | 不能 |
|---|---|
| 孤立堆对象自动释放（ARC） | 不级联释放子节点（不区分所属权） |
| 释放点检测悬挂（in>0） | 不反向失效（目标死不自动置 nil 引用者） |
| 释放点检测未清理出边（out>0）并定位源码 | 不保证无泄漏（环需手工断边、忘记释放照漏） |
| 全 `T@` 闭包内安全 | `T@` 转裸 `T&` 后退回纯 C，不可检测 |

## 12. 待实现注意（编译器侧硬约束）

1. **重新赋值先拆旧边**（§4.1）；所有出口（`return`/`goto` 出域/`break`）都要插拆解；
   禁止 `goto` 跳过域起始的 ref-init。
2. **域退出两阶段**（先拆边后断言），同域互指才不误报。
3. **禁止经裸 base 写胖成员**；导出到 op.h、会跨 C ABI 的结构体**禁含 `T@`**
   （含 `T@` 成员的结构体被 C 侧 memcpy/fread 会让 tar 成垃圾地址 → 下次赋值写坏内存）。
4. **胖目标地址稳定**：不得把按值塞进可扩容容器的对象当胖目标。
5. **`T@` 数组**退域遍历逐个 dec；`sizeof` 按 24B/元素，与 C 指针数组不同（已认）。
